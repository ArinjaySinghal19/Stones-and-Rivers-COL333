# Weight Training System

This directory contains a weight training system for optimizing the heuristic weights used by the Stones and Rivers bot using **hill climbing** with **random perturbations**.

## Overview

The system uses local search (hill climbing) to find better weights for the heuristic evaluation function. Key features:

- **Hill Climbing Algorithm**: Iteratively improves weights by accepting better configurations
- **Random Perturbations**: Neighbors are generated by perturbing 5-10% of weight values
- **Unit Norm Constraint**: All weight vectors are normalized to unit length (L2 norm = 1)
- **No Negative Weights**: Ensures all weights remain non-negative
- **Comprehensive Testing**: Full test suite verifies correctness

## How It Works

### Weight Vector

The `WeightVector` class represents a weight configuration with 12 weights:

1. `vertical_push` - Vertical push heuristic
2. `pieces_in_scoring_attack` - Offensive scoring pieces
3. `horizontal_attack_self` - Horizontal attack for self
4. `inactive_self` - Penalty for inactive pieces
5. `pieces_blocking_vertical_self` - Penalty for blocking own vertical
6. `horizontal_base_self` - Base horizontal value for self
7. `horizontal_negative_self` - Horizontal negative for self
8. `pieces_in_scoring_defense` - Defensive scoring pieces
9. `pieces_blocking_vertical_opp` - Penalty for blocking opponent vertical
10. `horizontal_base_opp` - Base horizontal value for opponent
11. `horizontal_attack_opp` - Horizontal attack for opponent
12. `inactive_opp` - Penalty for opponent inactive pieces

**Key Properties:**
- All weights are normalized to unit norm: ||w|| = 1
- No negative weights allowed
- Perturbations maintain normalization

### Hill Climbing Algorithm

```
1. Start with current weights W
2. Evaluate fitness F(W)
3. For each iteration:
   a. Generate neighbor W' by perturbing W (5-10%)
   b. Evaluate fitness F(W')
   c. If F(W') > F(W):
      - Accept W' (set W = W')
   d. Otherwise:
      - Reject W' (keep W)
4. Return best weights found
```

### Perturbation

Perturbation creates a neighbor by:
1. Generating a random direction vector
2. Scaling it to 5-10% magnitude
3. Adding to current weights
4. Taking absolute value (no negatives)
5. Normalizing to unit norm

### Fitness Evaluation

**Note**: The current implementation uses a **placeholder fitness function** based on weight balance. 

**To use real game evaluation**, you need to implement the `evaluate_fitness` method in `WeightTrainer` class to:

1. Update weights in `heuristics.h`
2. Compile the C++ code
3. Play games (self-play or against baseline)
4. Return win rate or ELO-based score

Example pseudocode:
```python
def evaluate_fitness(self, weights):
    # Update C++ weights
    self.update_cpp_weights(weights)
    
    # Compile
    self.compile_cpp()
    
    # Play games
    wins = 0
    for _ in range(self.games_per_eval):
        result = play_game()  # Implement this
        if result == "win":
            wins += 1
    
    return wins / self.games_per_eval
```

## Implementation Details

### Unit Norm Constraint

All weights are stored as normalized vectors with L2 norm = 1:

```python
norm = sqrt(w1² + w2² + ... + w12²) = 1
```

**Why?** This prevents weights from growing unbounded and ensures fair comparison during training.

**Denormalization:** For readability and C++ usage, weights are scaled by 100 when exported.

### Perturbation Formula

```python
perturbation_ratio = random(0.05, 0.10)  # 5-10%
direction = random_unit_vector()
perturbation = direction * perturbation_ratio * ||W||
W_new = normalize(abs(W + perturbation))
```

### No Negative Weights

After perturbation, absolute value ensures all weights remain non-negative:
```python
new_weights = abs(weights + perturbation)
```

This is important because negative weights could invert the meaning of heuristics.

## Theory: Why Unit Norm?

**Problem**: Without constraints, weights can grow unbounded during optimization.

**Solution**: Normalize to unit norm (L2 = 1).

**Benefits**:
1. **Scale invariance**: Heuristic value h(s) = w·f(s) is scale-invariant
2. **Fair comparison**: All weight vectors are on same scale
3. **Prevents overflow**: Weights stay bounded
4. **Geometric interpretation**: Weights lie on unit hypersphere

**Trade-off**: We lose absolute magnitude information, but relative proportions (what matters for decision-making) are preserved.

## Performance Tips

1. **Parallel Evaluation**: Evaluate multiple weight configurations in parallel
2. **Warm Start**: Start from good known weights instead of random
3. **Adaptive Perturbation**: Reduce perturbation ratio over time
4. **Early Stopping**: Stop if no improvement for N iterations
5. **Checkpointing**: Save intermediate results regularly
