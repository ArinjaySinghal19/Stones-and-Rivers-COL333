# Weight Training System - Quick Reference

## Overview

A complete weight training system for the Stones and Rivers bot using hill climbing with 5-10% random perturbations and unit norm constraint.

## What Was Created

### Core Files

1. **`train_weights.py`** - Main training script
   - Hill climbing algorithm
   - Random perturbations (5-10%)
   - Unit norm constraint (||w|| = 1)
   - No negative weights
   - Saves results to JSON and C++ format

2. **`test_weight_training.py`** - Comprehensive test suite
   - 16 tests covering all functionality
   - Tests normalization, perturbation, hill climbing
   - All tests pass ✓

3. **`demo_weight_training.py`** - Interactive demonstration
   - Shows weight operations
   - Demonstrates normalization
   - Simple hill climbing example

4. **`apply_weights.py`** - Apply trained weights to heuristics.h
   - Updates C++ code with new weights
   - Creates backup automatically

5. **`WEIGHT_TRAINING_README.md`** - Full documentation

## Weights Being Tuned

The system tunes these 12 weights from `heuristics.h`:

1. `vertical_push` - Vertical push heuristic
2. `pieces_in_scoring_attack` - Offensive scoring pieces
3. `horizontal_attack_self` - Horizontal attack for self
4. `inactive_self` - Penalty for inactive pieces
5. `pieces_blocking_vertical_self` - Penalty for blocking own vertical
6. `horizontal_base_self` - Base horizontal value for self
7. `horizontal_negative_self` - Horizontal negative for self
8. `pieces_in_scoring_defense` - Defensive scoring pieces
9. `pieces_blocking_vertical_opp` - Penalty for blocking opponent vertical
10. `horizontal_base_opp` - Base horizontal value for opponent
11. `horizontal_attack_opp` - Horizontal attack for opponent
12. `inactive_opp` - Penalty for opponent inactive pieces

def evaluate_fitness(self, weights: WeightVector) -> float:
    # Update weights in C++ file
    self.update_cpp_weights(weights)
    
    # Compile

        return -float('inf')

## Theory

### Why Unit Norm?

**Problem**: Without constraints, weights can grow unbounded.

**Solution**: Normalize to ||w|| = 1 (unit norm).

**Benefits**:
- Scale invariance: h(s) = w·f(s) scale-invariant
- Fair comparison: all weight vectors same scale
- Prevents overflow: weights stay bounded
- Geometric: weights on unit hypersphere

### Why 5-10% Perturbation?

- Too small: slow convergence
- Too large: random search
- 5-10%: good balance for local search

### Why Hill Climbing?

- Simple and effective
- No hyperparameters (unlike simulated annealing)
- Easy to understand and debug
- Good for local optimization
